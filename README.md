# 🚀 Portfólio de Qualidade: Projeto NaSalinha

Bem-vindo ao repositório de Auditoria de Qualidade do projeto **NaSalinha**. [cite_start]Este projeto documenta a jornada de 7 semanas para garantir a estabilidade e a experiência do usuário através de testes rigorosos[cite: 4, 9, 65].

---

## 📝 Descrição do Projeto
[cite_start]O objetivo deste repositório é apresentar uma auditoria completa de QA, identificando falhas técnicas e garantindo que as funcionalidades core atendam aos requisitos de negócio estabelecidos[cite: 9, 10].

## 🛠️ Tecnologias e Ferramentas
* [cite_start]**Ambiente:** Docker (para execução do sistema localmente)[cite: 68].
* [cite_start]**Testes de API:** [Inserir: Postman ou Insomnia][cite: 8].
* [cite_start]**Gestão/Documentação:** [Inserir: GitHub Issues e Projects ou pasta /docs][cite: 17, 18].
* [cite_start]**Foco Técnico:** Testes Funcionais, Validação de API e Regressão[cite: 45].

## 🎯 Áreas Core Testadas
[cite_start]De acordo com os requisitos, o foco da auditoria recai sobre[cite: 19, 20]:
1.  [cite_start]**Autenticação JWT:** Validação de login e gestão de acessos[cite: 21].
2.  [cite_start]**Check-in por Foto:** Upload de provas e validação de mídia[cite: 22].
3.  [cite_start]**Sistema de Pontos:** Cálculos, persistência em banco e ranking[cite: 23].

## 📂 Estrutura de Testes
[cite_start]Para cada uma das áreas acima, foram elaborados[cite: 28, 29]:
* [cite_start]**Casos Funcionais:** Foco na interface e comportamento esperado na tela[cite: 30, 46].
* [cite_start]**Casos de API:** Validação de endpoints, regras de negócio no backend e Status Codes[cite: 31, 84, 85].
* [cite_start]**Testes de Regressão:** Simulação de correções de bugs para garantir que o fluxo principal permanece estável[cite: 33, 48].

## 📈 Relatório de Testes Final
* [cite_start]**Métricas de Execução:** Total de casos planejados vs. executados[cite: 42].
* [cite_start]**Status:** Quantidade de testes que passaram (Pass) vs. falharam (Fail)[cite: 43].
* [cite_start]**Severidade:** Distribuição entre bugs Críticos, Maiores e Menores[cite: 44].

## 🚀 Como Executar este Projeto
1.  Clone este repositório.
2.  [cite_start]Suba o ambiente do projeto original utilizando o **Docker**[cite: 68].
3.  [cite_start]Acesse os arquivos na pasta `/docs` para visualizar os casos de teste detalhados[cite: 18].
4.  [cite_start]Importe a **Coleção de API** (arquivo JSON) no seu Postman/Insomnia para validar os endpoints[cite: 15, 54].
